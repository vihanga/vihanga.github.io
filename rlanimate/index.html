RLAnimate

Reinforcement Learning for Virtual Human Animation

A model-based, data-driven reinforcement learning framework for virtual character animation that enables a single agent to portray multiple types of interactive human-like behaviours.

Vihanga Gamage
Technological University Dublin
Supervisors: Dr. Cathy Ennis · Dr. Robert Ross

## Output Demonstrations ### Animation Sequences

Star Jump — Model-based RL Agent

The star jump was used as a kinematic-feasibility baseline — a non-social behaviour that tests whether reinforcement learning can coordinate multiple joints simultaneously. The agent learned this behaviour from basic animation signals without any physics engine feedback.

Star jump output sequence — five frames showing coordinated multi-joint movement produced entirely through reinforcement learning.

Pointing — Tracking a Dynamic Target

Pointing served as the first social gesture in the research progression. Unlike the star jump, pointing is a communicative act — its success depends on whether an observer perceives the character as directing attention toward a specific location. The agent generates pointing animation frame by frame, enabling it to track moving targets in real time.

Pointing output sequence — the character directs gaze and arm toward a spatial target (red sphere), adapting dynamically to its position.

### Gaze and Pointing Coordination Natural human behaviour involves constant transitions between active gestures and rest. The RLAnimate agent learns these transitions implicitly through the dynamics model, producing smooth movement into and out of gestures.

Active gaze-and-point gesture — the character orients body, head, and arm toward the target.

Behaviour Flexibility

RLAnimate agents accept continuous control parameters that modulate how a behaviour is performed. The exaggeration parameter varies the amplitude from subtle to highly exaggerated — produced by a single trained agent, not by switching between animation clips.

Behaviour flexibility through waving exaggeration. Top row: left-arm waving. Bottom row: right-arm waving. Each progresses from subtle to highly exaggerated movements.

## Framework Architecture RLAnimate enables a single virtual character to portray multiple types of human-like social behaviours. Each behaviour is controlled through uniform input parameterisation, and the agent generates animation frame by frame, responding dynamically to changing inputs. The framework separates training from operation. Training uses compact motion capture datasets to learn how animation dynamics evolve over time. At runtime, the learned model plans animation sequences on the fly, enabling responsive behaviour without replaying stored clips.

RLAnimate training and operation pipeline. During training (left), motion capture data and behavioural objectives learn a latent dynamics model. During operation (right), the learned model drives online planning to generate animation in real time.

Architecture Iterations

The framework was developed through four architecture iterations, each addressing specific limitations:

A1 — Initial RL-based Animation Control The A1 architecture introduced the core RLAnimate design: a mathematical framework that separates what the agent should portray (the objective), how the behaviour evolves over time (the dynamics), and what joint rotations produce the animation (the output). This separation allows one agent to learn multiple behaviours without retraining the animation generation pipeline.

A2 — Artefact Elimination The A2 iteration addressed occasional unnatural motion artefacts by splitting the latent space into portrayal-specific and behaviour-specific components. The agent learns to maintain smooth, natural motion while still responding flexibly to changing behavioural objectives.

A3 — Quaternion Representations and Finger Animation Introduced quaternion rotation representations and a dedicated animation dynamics model for fine-grained finger animation control.

A4 — Speech-driven Beat Gesture Generation Extended the framework for speech-driven conversational gestures with realism regularisation: rotation restrictions, collision detection, smoothness constraints, and adversarial discrimination.

A1 agent architecture — Portrayal Model, Behaviour Model, and Animation Model coordinated through a learned latent dynamics space.

A2 agent architecture — separate portrayal and behaviour latent dynamics spaces eliminate motion artefacts.

Technical Details — Component Architectures

Latent Dynamics Model

At the heart of each RLAnimate agent is a learned dynamics model that captures how animation states transition over time. Rather than using hand-crafted reward functions or physics simulation, the model learns temporal patterns directly from motion capture data.

Latent dynamics model — the hidden state (h) and behaviour state (b) evolve over time, conditioned on observations, descriptions, and previous actions.

Portrayal Model

The Portrayal Model encodes the current state of the animation into a compact latent representation. The dual-output design (mean and variance) captures both the most likely state and the uncertainty around it.

Portrayal Model neural network — observation passes through fully-connected layers, splitting into mean and variance parameters.

A2 Portrayal Model — GRU recurrent cell maintains temporal continuity with prior and posterior pathways.

Animation Model

The Animation Model translates the agent’s internal latent state into joint rotations applied to the virtual character. The beta-distribution parameterisation constrains outputs to valid rotation ranges.

A1 Animation Model — three fully-connected layers producing joint rotation outputs.

A2 Animation Model — draws on both latent dynamics spaces with expanded capacity (2000-unit layers).

Behaviour Model

The Behaviour Model captures how the character’s intended action evolves over time. The GRU recurrent cell maintains a memory of the behaviour trajectory, enabling smooth transitions and temporally coherent animation.

A2 Behaviour Model — GRU cell with prior and posterior distributions over the behaviour latent state.

## Key Results
p > 0.05

RLAnimate A4 output rated statistically indistinguishable from motion capture ground truth across all evaluation criteria.

p < .001

Significantly outperforms existing state-of-the-art (Gesticulator) across human-likeness, speech reflection, and audio synchronisation.

28 Participants

Rigorous perceptual evaluation using the GENEA Challenge protocol — the standard methodology for gesture generation assessment.

5ms / frame

Real-time generation — animation produced at interactive rates suitable for games and real-time applications.
